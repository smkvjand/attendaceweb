Firebase Setup
Connect to your existing project — same DB as the Android app
This website uses the exact same Firebase project as your Android app — same Firestore, same Auth users, same data. No migration needed.
Connect to Firebase (5 steps)
1

Open your Firebase project

Go to console.firebase.google.com and open the same project used for the Android app.

2

Register a Web App

Project Settings → Your apps → click the Web icon </> → give it a nickname → Register → copy the firebaseConfig object shown.

3

Paste config into this file

Scroll to the bottom of index.html, find the PASTE YOUR FIREBASE CONFIG HERE comment, and replace the placeholder:

const firebaseConfig = { apiKey: "AIza...", authDomain: "your-app.firebaseapp.com", projectId: "your-app-id", ... };
4

Update Firestore Rules

Firebase Console → Build → Firestore → Rules tab → paste:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }

Click Publish.

5

Add authorized domain

Firebase Console → Authentication → Settings → Authorized domains → Add → paste your GitHub Pages URL: your-username.github.io

Deploy to GitHub Pages
1

Create a GitHub repo

github.com → New repository → name: attendease → Public → Create repository.

2

Upload this file

In the repo, click "Add file" → "Upload files" → drag and drop index.html → Commit changes.

3

Enable GitHub Pages

Repo → Settings → Pages → Source: Deploy from branch → Branch: main → folder: / (root) → Save.

4

Done! Site is live at:

https://your-username.github.io/attendease
